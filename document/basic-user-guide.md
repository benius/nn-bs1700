# NN-BS1700 入門使用者指南

這份指南提供給第一次使用 `cooking-with-nn-bs1700` 的使用者。提問時只要提供「食譜名稱」與「主要食材」；保存狀態、份量、厚度、口感、模式、配件等條件都可省略。條件不完整時，skill 會依現有資料說明採用的假設，或分別列出生鮮、冷藏與冷凍等適用選項。

本頁的提示詞只使用 `cooking-with-nn-bs1700`，不需要搭配其他 skill。若想精確指定所有烹調條件、比較多種模式，或把料理流程回存成食譜，請改用[進階使用者指南](advanced-user-guide.md)的專業完整提示詞範本。

## Skill 名稱前的符號

下列範本使用 Codex 的 `$cooking-with-nn-bs1700` 寫法。不同 agent 可能改用 `/cooking-with-nn-bs1700`，或不使用任何符號而直接以自然語言指定 skill。若目前的 agent 不支援 `$`，把第一行改成「請使用 cooking-with-nn-bs1700 skill」即可。各平台的詳細差異請參考[進階使用者指南的 skill 叫用方式](advanced-user-guide.md#先確認平台的-skill-名稱與叫用方式)。

## 最精簡提示詞

只填兩項必填資料，就可以開始：

```text
請使用 $cooking-with-nn-bs1700，提供 Panasonic NN-BS1700 可以直接照著操作的料理流程。

食譜名稱：【必填，例如：蒜香烤雞腿】
主要食材：【必填，例如：去骨雞腿肉】

其他烹調條件未提供。請依 skill 的規則說明假設或列出適用選項，並告訴我準備方式、配件與位置、面板設定、首輪時間、完成判定，以及不足時如何補救。
```

## 可加入選填條件的入門範本

只有「食譜名稱」與「主要食材」一定要填。其他欄位知道多少填多少；不知道時可以刪除整行、留白，或填「不確定」。

```text
請使用 $cooking-with-nn-bs1700，為 Panasonic NN-BS1700 提供一套可以直接照著操作的料理流程。

【必填】
食譜名稱：【料理名稱】
主要食材：【主要食材】

【選填；可整段省略】
起始狀態：【生鮮／冷藏／冷凍】
份量：【重量、數量或人數】
最厚處：【厚度；是否帶骨或填餡】
目標口感：【濕潤／焦香／酥脆／全熟／回溫等】
偏好模式：【微波／蒸／烘烤／燒烤／由你推薦】
現有配件或容器：【不知道可不填】
其他限制：【調味、可用時間、不能使用的配件等】

沒有提供的條件，請依 cooking-with-nn-bs1700 的規則明確說明假設或列出不同選項，不要把推算值說成確定的原廠值或實測最佳值。先給最推薦的一套流程，內容包含：
1. 食材準備與需要的解凍方式。
2. 配件、容器、盤面、層位與是否預熱。
3. 每一段的面板模式、功率或溫度、首輪時間，以及翻面、攪拌或加入蒸氣的時機。
4. 中心溫度或可觀察的完成判定。
5. 未熟、不上色或太乾時的短段補救方式。

若某種做法不適合這項食材或 NN-BS1700，請直接說明並提供安全替代方案。這次只回答料理方法，不修改任何檔案。
```

## 簡單範例

以下範例刻意不提供保存狀態、份量、厚度與配件，agent 仍應依 skill 規則提出可操作的選項：

```text
請使用 $cooking-with-nn-bs1700，提供 Panasonic NN-BS1700 可以直接照著操作的料理流程。

食譜名稱：蒜香烤雞腿
主要食材：去骨雞腿肉

其他烹調條件未提供。請說明採用的假設或列出適用選項，先推薦一套做法，再寫出準備、配件與位置、面板設定、首輪時間、完成判定及補救方式。
```

如果已經知道食材溫度、重量、厚度、目標口感和手邊配件，填入選填欄位能讓建議更精準；但這些資料不是使用入門範本的前提。需要控制來源查找、模式比較、食譜建檔或實測升級時，再使用[進階使用者指南](advanced-user-guide.md)。
